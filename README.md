# Manim AI Animation Generation and Critical Evaluation

## Overview

This project explores the use of Google's Gemini API to automatically generate Manim animations for mathematical concepts. The objective is to evaluate the quality of AI-generated animation code, identify its strengths and weaknesses, and critically assess the resulting visualizations.

The assignment consists of two tasks:

1. Visual Proof of the Pythagorean Theorem
2. Fourier Series Decomposition Visualization

This repository contains:

- Prompt files used with Gemini
- Python code used to interact with the Gemini API
- Raw Manim code generated by Gemini
- Rendered animation outputs
- Critical analysis of the generated scenes

# Question 1 – Visual Proof of the Pythagorean Theorem

## Prompt Used

```text
Create a Manim Scene called PythagoreanTheorem.

Create a visual proof of the Pythagorean theorem.

Use a right triangle with squares on all three sides.

Demonstrate geometrically why the area of the square on the hypotenuse equals the sum of the areas of the other two squares.

Use clear animations and explanatory text.

Keep all objects visible.

Use only Text() for text.

Return only runnable Python code containing one Scene class.
```

## Running the Scene

```bash
manim -pql pythagoras.py PythagoreanTheorem
```

## Critical Analysis

### 1. Inward-Facing Hypotenuse Square (Geometric Overlap)

**Issue**

At approximately 0:08 in the animation, the square representing the area of the hypotenuse (c²) is drawn in the wrong direction.

**Impact**

This obscures the underlying geometry and makes the initial construction difficult to interpret.

**Corrective Action**

The hypotenuse square should be constructed outward so that all three squares remain visually distinct.

### 2. Floating and Detached Hypotenuse Label

**Issue**

The label "c" appears noticeably separated from the hypotenuse and remains visible at the end of the animation.

**Impact**

This weakens the visual connection between the label and the side it represents.

**Corrective Action**

Anchor the label closer to the midpoint of the hypotenuse and remove it with the diagram.

### 3. Severe Text-on-Diagram Overlap (Layout Overflow)

**Issue**

Several equations overlap the geometric proof diagram during the derivation.

**Impact**

The screen becomes cluttered and difficult to read.

**Corrective Action**

Use a split-screen layout for geometry and algebra.

### 4. Loss of Mathematical Abstraction

**Issue**

The derivation substitutes specific values (3 and 4) despite claiming the proof works for all right triangles.

**Impact**

The animation becomes a numerical demonstration rather than a general proof.

**Corrective Action**

Keep the derivation symbolic throughout.

### 5. Poor Typographic Formatting for Mathematics

**Issue**

Mathematical expressions are displayed using plain text rendering.

**Impact**

The notation appears less polished than professionally typeset mathematics.

**Corrective Action**

Use mathematical typesetting where assignment requirements permit.

## Key Findings

The quality of the generated output depended heavily on prompt design.

A higher-level prompt encouraged Gemini to generate a genuine geometric proof, but it also introduced visual and implementation issues that required human review.
